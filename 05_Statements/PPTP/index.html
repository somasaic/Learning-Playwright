33 - Basic If/Else Statement

33_Statement.js If/Else

The most fundamental conditional statement in JavaScript. If the condition evaluates to true, the first block executes; otherwise, the else block runs.

let age = 20;

if (age > 18) {
    console.log("Yes, Goa");
} else {
    console.log("Go home!");
}

34 - If/Else If Chain (Grading System)

34_If_else_If.js Else If

An else if chain allows checking multiple conditions in sequence. This example implements a grading system: A (90+), B (80+), C (70+), D (60+), or F (below 60). Only the first matching condition's block executes.

let score = 78;

if (score >= 90) {
    console.log("Grade: A — Excellent");
} else if (score >= 80) {
    console.log("Grade: B — Good");
} else if (score >= 70) {
    console.log("Grade: C — Average");
} else if (score >= 60) {
    console.log("Grade: D — Below Average");
} else {
    console.log("Grade: F — Fail");
}

35 - Real-World Example: Role-Based Access

35_REAL_LIVE_Example.js Nested If

A practical example showing nested if/else for role-based access control. First checks if the user is logged in, then checks their role (admin, editor, viewer) to determine access level. Demonstrates how real web applications handle authorization.

let isLoggedIn = true;
let userRole = "editor";
// app.vwo.com -> viewer, editor or admin
// viewer = limited view
// editor can edit and view
// admin can do all the things

if (isLoggedIn) {
    if (userRole === "admin") {
        console.log("admin can do all the things");
    } else if (userRole === "editor") {
        console.log("Welcome Editor — Edit access granted.");
    } else if (userRole === "viewer") {
        console.log("Welcome Viewer — Read-only access.");
    } else {
        console.log("No idea which role you are !");
    }
} else {
    console.log("You are not logged in!!")
}

36 - API Status Code Handling

36_API_IF_ELSE.js API Testing

A common pattern in API testing: checking HTTP status codes with if/else. Demonstrates handling status code 200 (success) and 404 (not found). Note: uses == which allows type coercion -- "404" (string) matches with loose comparison.

let statusCode = 200; // APIs are working fine

if (statusCode == 200) {
    console.log("Working fine!")
} else if (statusCode == "404") {
    console.log("not found!");
} else {
    console.log("Not matching status code!")
}

37 - Truthy and Falsy Values

37_IQ_IF_ESLE.js Truthy/Falsy

JavaScript has truthy and falsy values. Truthy: non-empty strings, non-zero numbers, objects {}, arrays []. Falsy: empty string "", null, undefined, NaN, 0. Understanding this is crucial for writing correct conditional checks.

//true
if ("hello") console.log("String is truthy"); // "hello" = truthy
if (42) console.log("Number is truthy");
if ({}) console.log("Empty object is truthy!");
if ([]) console.log("Empty array is truthy!");

// false results
if ("") console.log("Won't print");  // "" -> falsy result
if (null) console.log("Won't print");
if (undefined) console.log("Won't print");
if (NaN) console.log("Won't print");
if (0) console.log("Won't print");

38 - Logical Operators in If/Else

38_Logical_Op_IF_ELSE.js Logical + If

Combining logical operators (&&, !) with if/else for complex conditions. This login example checks if the username and password match AND the account is not locked. The ! operator negates the isAccountLocked flag.

let username = "Dev";
let password = "secure123";
let isAccountLocked = true;

if ((username === "Dev" && password === "secure123") && !isAccountLocked) {
    console.log("Allowed to enter");
} else {
    console.log("not allowed to enter");
}

39 - Short If Syntax (No Braces)

39_IQ.js Syntax

If the if block contains only a single statement, you can omit the curly braces. While valid, this is generally discouraged in production code as it can lead to bugs when adding more lines later.

let x = 10;
if (x > 5)
    console.log("x is big");
